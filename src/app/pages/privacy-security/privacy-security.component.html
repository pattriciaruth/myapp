Privacy & Security

This inventory app was built for learning purposes in PROG2005. It does not use any backend or database. All data is stored in memory only while the page is open.

Data storage

  • Items are kept in an in-memory TypeScript array inside an Angular service.
  • When the page is refreshed or closed, all inventory data is lost.
  • No information is sent to a server or external API.

Input validation & safety

  • Required fields are checked before items are added.
  • Numeric values such as quantity, price and stock level are restricted to sensible ranges.
  • Angular templates automatically escape text when it is displayed, which helps reduce simple cross-site scripting risks.

Personal data

  • The system tracks products only. It does not store user accounts, passwords or customer details.
  • Because there is no login and no persistent storage, the privacy risks are very low.

In a real production system, a secure backend, authentication, HTTPS and stronger server-side validation would be required. For this assignment, client-side TypeScript and Angular validation are sufficient.
